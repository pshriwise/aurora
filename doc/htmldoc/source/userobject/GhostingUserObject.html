      GhostingUserObject

      The GhostingUserObject maintains local data structures of elements involved in both "Geometric" and "Algebraic" ghosting. This object is normally setup automatically through the (DisplayGhostingAction.md).

      Example Input Syntax:

      Creates ghosting maps that can be queried by external objects.

      Input Parameters

      • execute_onINITIALThe list of flag(s) indicating when this object should be executed, the available options include NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM, ALWAYS.

        Default:INITIAL

        C++ Type:ExecFlagEnum

        Options:NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM, ALWAYS

        Description:The list of flag(s) indicating when this object should be executed, the available options include NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM, ALWAYS.

      • pidsThe PID(s) to see the ghosting for

        C++ Type:std::vector<unsigned int>

        Options:

        Description:The PID(s) to see the ghosting for

      • prop_getter_suffixAn optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

        C++ Type:MaterialPropertyName

        Options:

        Description:An optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.
